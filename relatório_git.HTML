Relatório - Aula 19/10

Repositório – é um espaço onde podemos armazenar arquivos específicos de um projeto, por exemplo. Você pode acessar por uma URL específica também.

  • Git clone: linha de comando usada para selecionar um repositório que já existe e criar uma cópia do repositório;
  • ssh -T git@github.com: se conecta com o GitHub por SSH (Secure Shell);
  • Git status: verifica o estado do arquivo, se tem algum que foi modificado ou removido;
  • Git add: adiciona o arquivo selecionado ou o atualiza, se for adicionado “.” logo após, ele irá adicionar ou atualizar todos os arquivos que estão faltando;
  • Git commit -m: são alterações que foram feitas nos arquivos, possui o que foi alterado, seu autor e uma mensagem de resumo;
  • Git push: é usado para compartilhar as modificações;
  • Git pull: é usado para baixar arquivos de repositórios remotos e fazer a atualização para o repositório local para que o conteúdo seja o mesmo;
  • Git log: mostra que alterações vêm sendo feitas em uma branch;
  • Git branch: deixa o usuário criar, listar, renomear e excluir ramificações;
  • Git checkout: é usado para mudar de branch, ou voltar em algum arquivo;
  • Git merge: deixa tudo em apenas uma branch;
